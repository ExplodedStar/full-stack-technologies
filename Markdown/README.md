## Введение

***Markdown***  — облегчённый язык разметки, созданный с целью написания максимально читаемого и удобного для правки текста, но пригодного для преобразования в языки для продвинутых публикаций (HTML, Rich Text и других).

Из плюсов Markdown можно выделить:
- **Легко изучаем**
- **Легкое преобразование** в другие языки
- Отлично **подходит для написания документаций и статей**

Пример простой Markdown страницы на входе:

```markdown
### Заголовок статьи

Меню
- Раздел 1
	- Подраздел 1.1
	- Подраздел 1.2
- Раздел 2
- Раздел 3

***Программирование*** — процесс и искусство создания компьютерных программ с помощью языков программирования.
```
На выходе мы получим:

### Заголовок статьи

Меню
- Раздел 1
	- Подраздел 1.1
	- Подраздел 1.2
- Раздел 2
- Раздел 3

***Программирование*** — процесс и искусство создания компьютерных программ с помощью языков программирования.

В следующих главах мы разберём.

## Заголовки 

**Создание заголовков производится путём помещения знака решетки перед текстом заголовка**. Количество знаков «#» соответствует уровню заголовка. Markdown предоставляет **6 уровней заголовков**.

```markdown
# H1
## H2
### H3
#### H4
##### H5
###### H6
```

# H1
## H2
### H3
#### H4
##### H5
###### H6

## Списки
В Markdown существуют несколько видов списков: нумерованный, маркированный и зачеркнутый.
### Нумерованный список
Нумерованный список создается путем _**добавления номера и точки после него**_. Так же вы можете создавать вложенные списки используя табуляцию.
> Порядок номера не имеет значения так как он добавляется автоматически.
```markdown
1. Пункт 1
2. Пункт 2
3. Пункт 3
   1. Подпункт 3.1
   2. Подпункт 3.2
```

1. Пункт 1
2. Пункт 2
3. Пункт 3
   1. Подпункт 3.1
   2. Подпункт 3.2

### Маркированный список
Вы можете **_использовать - (дефисы) или * (звездочки)_** для создания маркированного списка.
```markdown
* Пункт 1
* Пункт 2
  * Подпункт 2.2
  * Подпункт 2.2
```

* Пункт 1
* Пункт 2
  * Подпункт 2.1
  * Подпункт 2.2

## Сcылки
Существует два пути создания ссылок.
```markdown
[Я inline-style ссылка](https://www.google.com)

[Я inline-style ссылка с заголовком](https://www.google.com "Google's Homepage")

[I'm a reference-style link][Arbitrary case-insensitive reference text]

[I'm a relative reference to a repository file](../blob/master/LICENSE)

[You can use numbers for reference-style link definitions][1]

Or leave it empty and use the [link text itself].

URLs and URLs in angle brackets will automatically get turned into links. 
http://www.example.com or <http://www.example.com> and sometimes 
example.com (but not on Github, for example).

Some text to show that the reference links can follow later.

[arbitrary case-insensitive reference text]: https://www.mozilla.org
[1]: http://slashdot.org
[link text itself]: http://www.reddit.com
```

## Цитаты
Для **создания цитаты используют > (знак больше)** и текст после него. Также вы можете создавать вложенные цитаты.
```markdown
> Это пример обычной цытаты
>> А это пример вложенной цитаты
```
> Это пример обычной цытаты
>> А это пример вложенной цитаты

## Горизонтальные линии
Существует несколько путей создания горизонтальных линий с помощью _**тройных: - (дефисов), * (звездочек) или _ (линий нижнего подчеркивания)**_.
```markdown

---

Дефисы (hyphens)

***

Звездочки (asterisks)

___

Линии нижнего подчёркивания (underscores)
```

---

Дефисы (hyphens)

***

Звездочки (asterisks)

___

Линии нижнего подчёркивания (underscores)

## Таблицы
Вы можете создавать таблицы, **разделяя их дефисами - (hyphens)** (_для первого ряда_), а **затем разделяя каждый столбец | (pipe)**:
```markdown
Первый заголовок | Второй заголовок
------------ | -------------
Содержимое ячейки 1 | Содержимое ячейки 2
Содержимое первой колонки | Содержимое второй колонки
```
На выходе мы получим:

Первый заголовок | Второй заголовок
------------ | -------------
Содержимое ячейки 1 | Содержимое ячейки 2
Содержимое первой колонки | Содержимое второй колонки
